What we announced at Build: What’s next for Visual Studio
At Build this year, we were excited to share where Visual Studio is headed next. Across these announcements, our focus is simple: helping professional developers ship high-quality code faster, with less friction, and with AI that earns its place in the workflow.
A lot of what we showed builds on an idea we’ve believed in for a while: code is an asset, not just an artifact. The tools around it should help you keep it healthy, correct, and easy to evolve as your codebase grows.
Agents that help you do the work
We opened with GitHub Copilot in Visual Studio, not just as a chat or completion experience, but as the beginning of something more capable. We’re moving toward agents that can participate more actively in the development lifecycle, helping with debugging, profiling, and testing alongside you.
This is not about replacing the tools you already rely on. It is about connecting them more effectively. The debugger, profiler, and test tools already provide deep insight, and agents can help turn that insight into action:
•	Identify issues faster
•	Explain what’s going on
•	Suggest concrete fixes
•	Help validate the results
Our goal is to make this feel natural inside Visual Studio, especially for professional C# and C++ developers working in large, complex codebases.
App modernization
This summer, we're bringing new capabilities to GitHub Copilot modernization, the integrated agent experience built into Visual Studio that helps you upgrade your applications to the latest .NET stack. You can now migrate Web Forms applications to Blazor for a modern, component-based web stack, and add Aspire to existing apps for cloud-ready observability and orchestration. The modernization agent assesses your project, builds a plan, and executes upgrades step by step, helping you improve performance and security without starting from scratch.
Skills that adapt to your project
Another important shift is how Copilot understands your project and the work you’re doing.
Today, many of the right capabilities already exist, but they are not always surfaced at the right moment.
We’re introducing Microsoft-authored skills that can be applied automatically based on your project type and the task at hand.
That means less prompting, less guesswork, and a more helpful experience overall. The right capabilities can show up when you need them, without requiring you to already know they exist.
Faster builds by catching issues earlier
Long builds that fail at the very end are still a common source of frustration.
Today, a build can still run even when there are already obvious errors waiting in the Error List.
We’re changing that flow so Visual Studio can check errors and warnings before the build starts.
It is a simple change, but it can save time and help you focus on the issues that matter instead of waiting on avoidable failures.
Making merge conflicts less painful
Merge conflicts are something every developer runs into, and they are rarely a good use of anyone’s time.
We’re working on AI-assisted conflict resolution to reduce how much manual effort these situations require.
The goal is not to auto-merge everything. It is to help you understand the conflict, make sensible decisions, and get moving again faster.
A more flexible AI foundation (BYOK)
We’re also rethinking how AI connects into Visual Studio more broadly.
Historically, these integrations have been limited to a small set of sanctioned endpoints.
We’re moving toward a BYOK (bring your own key or model) approach so you can use different AI models, whether they run locally or in the cloud.
That gives you more flexibility around performance, cost, and compliance based on the needs of your environment.
Built on the GitHub Copilot SDK
Underneath all of this is a more unified foundation.
Visual Studio is moving to the GitHub Copilot SDK as the foundation for its AI integration going forward.
That helps us move faster, stay aligned with the broader ecosystem, and bring new capabilities into Visual Studio sooner.
Where this is heading
If there’s one way to sum up this roadmap, it’s this:
We’re focusing on a set of meaningful improvements that remove friction from the inner loop and make day-to-day development feel better.
•	Code that compiles by default
•	Faster feedback before you build
•	Smarter handling of real-world pain points like merge conflicts
•	AI that works with your tools, not next to them
•	Flexibility in how you bring AI into your environment
Most importantly, all of this is designed to fit into the way you already use Visual Studio, not force you into a different workflow.
