# GitHub Copilot in Visual Studio: Agents That Debug, Profile, and Test

This breakout session explores how GitHub Copilot in Visual Studio helps professional developers work more effectively across debugging, profiling, testing, and broader development workflows.

## What this session covers

In this session, you will see how agent-powered experiences in Visual Studio can help you:

- investigate and resolve issues more efficiently during debugging
- analyze performance bottlenecks using profiling workflows
- improve confidence and speed when working with tests
- accelerate common development tasks with deeper GitHub Copilot integration
- explore upcoming capabilities such as modernization, skills, merge conflict assistance, bring your own key support, and the GitHub Copilot SDK

## What we announced at Build

This session highlights how GitHub Copilot is evolving in Visual Studio from inline assistance to broader agent-powered workflows that help remove friction from the inner development loop.

The focus areas covered in the session include:

- debugger, profiler, and test integrations that help developers move more quickly from problem discovery to resolution
- app modernization experiences that help developers update and improve existing codebases
- Microsoft-authored skills that bring more task-specific intelligence into developer workflows
- faster pre-build feedback to help catch issues earlier in the development cycle
- AI-assisted merge conflict resolution to reduce time spent reconciling changes
- bring your own key support for teams with specific model or platform requirements
- the GitHub Copilot SDK for building and extending custom agent experiences

## Key capabilities to watch

If you want to continue exploring the themes from this session, pay particular attention to how GitHub Copilot in Visual Studio can help across different stages of development:

- **Debugging:** investigate failures, reason about root causes, and suggest next diagnostic steps
- **Profiling:** identify performance bottlenecks and focus attention on the code paths that matter most
- **Testing:** understand failing tests, improve confidence in changes, and speed up validation workflows
